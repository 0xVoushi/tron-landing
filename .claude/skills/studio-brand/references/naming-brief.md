# Studio Naming Brief

## Naming Criteria

A good studio name should be:

### Must Have
- **Memorable** — sticks after one exposure
- **Spell-able** — no ambiguous spellings or phonetics
- **Domain-available** — .dev, .io, .studio, or .com
- **Short** — ideally 1-2 words, max 3 syllables per word
- **Unique** — not confused with existing studios/companies

### Nice to Have
- **SEO-friendly** — contains or implies "dev" / "studio" / "tech" / "labs"
- **International** — works across English and other languages
- **Visual** — easy to create a logo/icon for
- **Tonal fit** — matches the brand personality (technical + modern)

### Avoid
- Generic tech clichés (Synergy, Nexus, Apex, Quantum)
- Hard-to-spell words or unusual letter combinations
- Names that limit future scope (e.g., "Web3 Only Labs")
- Names already used by popular dev tools or frameworks

## Naming Approaches

### 1. Compound Word
Combine two meaningful words: **Pepper** + **Sec**, **Iron** + **Forge**, **Code** + **Craft**

### 2. Abstract / Invented
Create a new word with good phonetics: **Vercel**, **Supabase**, **Figma**

### 3. Metaphor / Concept
Use a concept that represents the work: **Forge**, **Scaffold**, **Catalyst**

### 4. Descriptor
Clear and functional: **[Name] Studio**, **[Name] Labs**, **[Name] Dev**

## Process

1. Generate 10-15 candidates using approaches above
2. Filter by domain availability (quick check)
3. Shortlist top 5
4. Score using evaluation matrix
5. Test in context: "Welcome to [Name]", "[Name] — Building the Future", "Built by [Name]"
6. Test in meta title: "[Name] — Web Development, AI & Web3 Studio"
7. Final pick
