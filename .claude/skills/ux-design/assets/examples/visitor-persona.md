# Visitor Persona: Startup CTO

## Profile

- **Name**: Alex (composite persona)
- **Role**: CTO / Technical Co-founder at early-stage startup
- **Company Size**: 2-10 people
- **Budget**: $20K-80K for initial product build
- **Technical Level**: High — can evaluate code quality and architecture
- **Decision Timeline**: 1-2 weeks

## Goals

1. Find a development partner who can ship an MVP in 4-8 weeks
2. Evaluate technical depth (are they actually senior engineers?)
3. Understand pricing model and timeline expectations
4. See relevant portfolio work

## Pain Points

1. Previous freelancer/agency delivered low-quality code
2. Worried about handoff quality and maintainability
3. Tight budget, needs transparent pricing
4. Doesn't want to explain basic technical concepts

## Behavior on Landing Page

- **Arrives via**: Google search ("fullstack development agency"), referral, or GitHub
- **First 5 seconds**: Scans H1 for relevance, checks for technical credibility signals
- **Next 30 seconds**: Scrolls to services, looks for specific tech stack mentions
- **Decision point**: Case studies with measurable results, or process section showing structure
- **Conversion trigger**: "Book a Discovery Call" CTA after FAQ addresses timeline/pricing concerns

## What Converts This Persona

- Specific tech stack mentions (not vague "modern technologies")
- Real metrics: "shipped in 6 weeks", "95+ Lighthouse score"
- Process clarity: "Week 1: Discovery, Week 2: Architecture..."
- Low-friction CTA: "15-minute call" not "full project brief"

## What Turns This Persona Away

- Stock photos, generic agency language
- No technical detail (just "solutions" and "innovation")
- No case studies or portfolio
- "Contact us" without any indication of what happens next
